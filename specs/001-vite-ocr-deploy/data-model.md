# Data Model: API Routes配信とViteフロント統合デプロイ

## Entities

### OCRリクエスト
- **Attributes**:
  - 画像ファイル群: 1〜10枚、各最大10MB（上限を超えるとバリデーションエラー）
  - メタ情報: エラー/成功メッセージ（クライアント提示用）
  - 処理結果: OCR抽出結果のJSON（スキーマはサーバ側で管理）
- **Rules/Validation**:
  - ファイル枚数・サイズの上限チェック
  - 必須フィールド: `images`（multipart）
  - エラー時は利用者が原因を特定できるメッセージを返却

### デプロイ構成
- **Attributes**:
  - フロント配信パス: `/ui/`
  - APIエンドポイント: `/api/ocr`, ヘルスチェック `/api/ocr/health`
  - 環境変数: `OPENAI_API_KEY`（必須）、`MOCK_OCR`（任意）、タイムアウト設定（任意）
  - ビルド手順: 単一コマンドでフロント成果物を同梱
- **Rules/Validation**:
  - 環境変数が未設定の場合、ヘルスチェックで検知しエラー表示する
  - フロントはデフォルトで同一ドメインを呼び出し、必要時のみ環境変数で上書え可能
